# Агент по разбору junior-вакансий / стажировок

Мини-агент, который помогает кандидату без опыта решить, на какие
стажировки и junior-вакансии стоит откликаться

## Короткое описание 

- **Проблема.** Junior тонет в списке вакансий: половина — это «junior» только на
  словах, часть — не то направление, не тот город, устаревшие или с зарплатой ниже
  порога. Руками это разбирать долго и субъективно
- **Pipeline.** `источник (CSV/JSON или реальный API trudvsem) → чтение критериев
  (criteria.md) → фильтр по направлению → детерминированный скоринг по правилам →
  вердикт LLM (apply/maybe/skip) с fallback и отсевом skip →
  Markdown-отчёт + лог → опциональная отправка в Telegram`.
- **Источники данных.** Mock-файлы `data/*` для воспроизводимости + 
  открытый API «Работа России» (`trudvsem`, без ключа, отдаёт настоящие ссылки).
  HeadHunter (`api.hh.ru`) тоже поддержал бы поиск, но сейчас требует OAuth-токен
  приложения (отдаёт 403 без авторизации), поэтому по умолчанию используется trudvsem.
- **Где LLM, где обычная логика.** Скоринг, фильтры, дедупликация, парсинг и
  обработка ошибок — это **обычная детерминированная логика** (прозрачные веса).
  **LLM** работает поверх и помогает *принять решение*: выносит вердикт и человеческое
  объяснение «почему подходит / что смущает». Без ключа агент не падает, а считает
  вердикт по правилам (порог по score).
- **Ограничения и риски.** Данные mock (никакого парсинга/обхода площадок); качество
  зависит от полноты полей в вакансии; веса скоринга подобраны эвристически; LLM может
  ошибаться — поэтому он лишь дополняет правила, а не заменяет их, и его вывод всегда
  сверяется с детерминированным разбором баллов в отчёте.
- **Самое сложное.** Сделать так, чтобы «агентность» была настоящей (помощь в решении),
  но при этом результат оставался воспроизводимым и устойчивым к пустым/битым/дублирующимся
  данным — отсюда гибрид «правила + опциональный LLM с fallback».

## Как запустить

Зависимостей нет (только Python 3.9+). LLM-слой опционален.

```bash
# базовый прогон на правилах (без сети, воспроизводимо)
python3 agent.py --input data/vacancies.csv --criteria criteria.md --top 5

# JSON вместо CSV
python3 agent.py --input data/vacancies.json --top 3

# РЕАЛЬНЫЕ вакансии из открытого API «Работа России» (trudvsem), без ключа
python3 agent.py --source trudvsem --top 5
python3 agent.py --source trudvsem -q "продуктовый аналитик" -q "data analyst" --limit 80

# включить LLM-вердикт (необязательно). Поддерживается любой OpenAI-совместимый
# провайдер: OpenRouter (есть бесплатные модели), OpenAI, DeepSeek, Ollama (локально).
# Пакет openai ставим в виртуальное окружение:
python3 -m venv .venv
./.venv/bin/python -m pip install -r requirements.txt
# ключ положи в .env (OPENROUTER_API_KEY=...) — бесплатный: https://openrouter.ai/keys
# и запускай через venv-питон, чтобы был доступен пакет openai:
./.venv/bin/python agent.py --source trudvsem --top 5
./.venv/bin/python bot.py
```

Результат — `reports/report.md`, лог действий — `reports/agent.log`.

### Отправка в Telegram

Доставка работает на стандартной библиотеке (без зависимостей). Нужны токен бота
(от [@BotFather](https://t.me/BotFather)) и chat id (узнать у [@userinfobot](https://t.me/userinfobot)).

```bash
# превью сообщения без отправки (токен не нужен)
python3 agent.py -i data/vacancies.csv --tg-dry-run

# реальная отправка: короткое summary + файл report.md
export TELEGRAM_BOT_TOKEN=123456:ABC...
export TELEGRAM_CHAT_ID=123456789
python3 agent.py -i data/vacancies.csv --telegram

# без вложения файла, только текст
python3 agent.py -i data/vacancies.csv --telegram --tg-no-file
```

Если токен/chat id не заданы — отправка тихо пропускается, агент не падает.

### Интерактивный Telegram-бот (триггер по сообщению)

Вместо ручного запуска можно поднять бота, который сам реагирует на сообщения:

```bash
export TELEGRAM_BOT_TOKEN=...   # или положи в .env
python3 bot.py
```

Команды в чате:
- `/start` или `/help` — справка с кнопками меню;
- `/anketa` — заполнить анкету (с кнопками для уровня, города, удалёнки и т.д.);
- `/me` — показать анкету и статус подписки;
- `/find продуктовый аналитик` — поиск по запросу (реальные данные trudvsem);
- `/subscribe` / `/unsubscribe` — ежедневный дайджест новых вакансий под анкету;
- просто текст (например `аналитик данных`) — тоже воспринимается как поисковый запрос.

В ответ бот присылает краткий разбор (apply/maybe/skip) и файл с полным отчётом.
Работает на long polling (`getUpdates`), без вебхуков и без внешних зависимостей.
Логика поиска/скоринга — общая с CLI (модуль `src/pipeline.py`).

**Постоянство данных.** Анкеты и подписки хранятся в `AGENT_DATA_DIR` (по умолчанию
`data/`) с атомарной записью. На хостинге примонтируй том и укажи путь в
`AGENT_DATA_DIR`, чтобы данные переживали редеплой (см. `DEPLOY.md`).

**Дайджест.** Фоновый воркер раз в сутки (`AGENT_DIGEST_INTERVAL_SEC`) ищет свежие
вакансии под анкету подписчиков и присылает только новые, не показанные ранее.

### Аргументы CLI

| Флаг | Назначение | По умолчанию |
|------|------------|--------------|
| `--input, -i` | CSV или JSON с вакансиями (для `--source file`) | `data/vacancies.csv` |
| `--source` | `file` или `trudvsem` (реальный API) | `file` |
| `--query, -q` | Поисковый запрос для trudvsem (можно несколько) | `продуктовый аналитик` |
| `--limit` | Сколько вакансий тянуть на запрос (trudvsem) | `50` |
| `--min-results` | Порог, ниже которого добираются fallback-запросы | `10` |
| `--criteria, -c` | Файл критериев пользователя | `criteria.md` |
| `--output, -o` | Куда сохранить отчёт | `reports/report.md` |
| `--top, -n` | Сколько вакансий в отчёте | `5` |
| `--today` | Дата отсчёта свежести (YYYY-MM-DD) | сегодня |
| `--log` | Файл лога | `reports/agent.log` |
| `--telegram` | Отправить отчёт в Telegram | выкл |
| `--tg-dry-run` | Показать Telegram-сообщение без отправки | выкл |
| `--tg-token` / `--tg-chat` | Креды (или env `TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID`) | env |
| `--tg-no-file` | Не прикреплять файл отчёта | выкл |

  trudvsem.py       # реальный источник: открытый API «Работа России» (без ключа)
  criteria.py       # парсинг criteria.md
  scorer.py         # детерминированный скоринг по правилам
  llm.py            # LLM-вердикт + детерминированный fallback
  report.py         # генерация Markdown-отчёта + summary для Telegram
  telegram.py       # Telegram Bot API: отправка + getUpdates (stdlib)

## Критерий успеха

Запуск на `data/vacancies.csv` отрабатывает без ошибок, корректно отбрасывает
1 дубль и 1 битую строку (из 14 строк → 12 валидных), формирует топ-5 с вердиктами
и объяснениями. На пустом/битом/несуществующем файле агент не падает, а выдаёт отчёт
с предупреждением и описанием проблемы.

## Надёжность (что обработано)

- пустой файл, несуществующий файл, битый JSON/CSV;
- строки без заголовка / без id / с пропущенными полями;
- дубликаты (по паре «заголовок + компания» и по URL);
- отсутствие `criteria.md` (берутся дефолты);
- отсутствие `OPENAI_API_KEY` или пакета `openai`, сетевые ошибки и кривой ответ LLM
  (везде fallback на правила);
- все шаги пишутся в лог.






# Вакансии — Telegram-бот для поиска junior-вакансий

Бот помогает найти подходящие стажировки и junior-вакансии: забирает реальные данные из API «Работа России», фильтрует их под твой профиль и присылает топ-5 с вердиктом — откликаться или нет.

---

## Как это работает

```
Анкета пользователя → API trudvsem → очистка данных → скоринг → LLM-вердикт → отчёт в Telegram
```

1. Пользователь заполняет анкету в боте: направление, навыки, город, зарплата
2. Бот запрашивает вакансии через официальный открытый API trudvsem.ru (без ключа)
3. Чистит данные: убирает дубли, заполняет пустые поля
4. Считает баллы по каждой вакансии по правилам (совпадение навыков, город, уровень, зарплата)
5. LLM читает топ вакансий и пишет вердикт: **apply / maybe / skip** с объяснением
6. Бот присылает разбор в чат + файл `report.md`

Если LLM недоступна — бот выдаёт результат по правилам, не падает.

## Ограничения

- Качество зависит от полноты данных на trudvsem — если компания не указала навыки или зарплату, бот может недооценить вакансию
- Если API trudvsem недоступен — новые вакансии не придут
- LLM может ошибаться, поэтому её вердикт всегда дополняется числовым score
